# プロジェクト概要: YouTube Learning Companion App

## 目的 (What & Why)
このプロジェクトの目的は、YouTube動画のトランスクリプト（字幕）を活用し、動画視聴、タイムスタンプ付き目次の探索、およびコンテンツに関する対話型チャットを融合させた最高の学習体験を実現するWebアプリケーションを開発することである [1]。

## エージェント構成 (How: マルチエージェントによる関心事の分離)
長期間の自律的な開発（ロングラン）と精度の維持のため、タスクを以下の2つの専門エージェントに分離する [2-5]。

1.  **イニシャライザーエージェント (Initializer Agent)**
    *   初期設定とタスクのオーケストレーションを担当する [2, 3, 6]。
    *   このタスクの全体的な機能リストを**JSON形式**で詳細に作成する（早すぎる完了を制限するため、機能要素を大量に書き出すこと）[2, 3, 5, 7]。
    *   Gitの初期化を行い、ファイル差分を追跡できるようにする [6]。
    *   作業ログ（`progress.txt`）の定義を行う [6]。

2.  **コーディングエージェント (Coding Agent)**
    *   イニシャライザーが作成した**機能リストからタスクを一つずつ選んで**実装する [2, 3, 6]。
    *   タスク完了後にはコミットログを作成し、次のタスクへ引き継ぐ [2, 3, 5]。

## 詳細情報の外部管理 (段階的開示)
LLMのコンテキストウィンドウを圧迫するのを避けるため、プロジェクト固有の詳細な情報（UIデザインのアイデア、API仕様、具体的な機能要件など）は`CLAUDE.md`に直接記述せず、`docs/`ディレクトリに格納し、必要な時に参照させる [8-11]。

## 技術スタック
- **フレームワーク**: Next.js (App Router)
- **認証**: NextAuth.js + Google Provider
- **スタイリング**: Tailwind CSS
- **データベース**: 未定（Prisma推奨）
- **API**: YouTube Data API v3

## コーディング規約
- コンポーネントは`src/components/`に配置
- API Routesは`src/app/api/`に配置
- 型定義は`src/types/`に配置
- ユーティリティ関数は`src/lib/`に配置
